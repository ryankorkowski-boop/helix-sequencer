XLIGHTS AUTO-SEQUENCER ONE-CLICK GUIDE

Dream Sequence Weaver now targets show-ready output by default. The maintained master profile still rides on the proven v27.3 engine, but every run now continues through audit, polish, variant generation, and optional auto-shortlisting before the final file is presented to you.

QUICK START

1. Put these files in one working folder:
   - the active project files, including `core/`, `tools/`, `xlights/`, and `main.py`
   - your xLights template sequence, usually `template.xsq`
   - your current xLights layout export, usually `xlights_rgbeffects.xml` or `xlights_rgbeffects.xbkp`
   - the audio file you want sequenced, for example `13.wav`

2. Optional but recommended:
   - drop any favorite high-scoring `.xsq` files into the workspace history folder so `--learn-from-my-xsqs` can learn your preferred palettes, densities, and prop behaviors
   - if you use Moises.ai, set `MOISES_API_KEY` or pass `--moises-api-key` and add `--use-moises`

3. Run the maintained one-click command:
   - `python312.cmd main.py --profile master -- --template template.xsq --audio 13.wav --no-prompt --polish --variants 3 --auto-shortlist --learn-from-my-xsqs`

4. Wait for the pipeline to finish. The engine now performs:
   - template and layout analysis
   - audio and stem-aware analysis
   - prop-aware effect placement
   - super-audit scoring and light automatic fixes
   - polish for hook emphasis, breathing fades, overlap cleanup, micro-timing, and color flow
   - optional multi-variant generation
   - optional auto-shortlisting of the best final result

5. Use the generated output directly:
   - the canonical `.xsq` is intended to be ready for show use
   - the `.report.json` explains quality, audit, shortlist, spatial flow, and coverage
   - the `.sequence_notes.txt` explains the polish actions and readiness summary

6. Open xLights only when you want optional artistic review or confidence verification.
   - manual cleanup is no longer the default expectation
   - in a normal successful run, you should not need a manual cleanup pass just to make the sequence usable

BEST PRACTICE COMMANDS

- Standard one-click render:
  `python312.cmd main.py --profile master -- --template template.xsq --audio 13.wav --no-prompt --polish --variants 3 --auto-shortlist`

- Learn from your own successful sequences:
  `python312.cmd main.py --profile master -- --template template.xsq --audio 13.wav --no-prompt --polish --variants 3 --auto-shortlist --learn-from-my-xsqs`

- Force Moises-aware stem analysis when credentials are available:
  `python312.cmd main.py --profile master -- --template template.xsq --audio 13.wav --no-prompt --polish --variants 3 --auto-shortlist --use-moises`

- Run the explicit compatibility profile:
  `python312.cmd main.py --profile v27.3 -- --template template.xsq --audio 13.wav --no-prompt --polish`

WHAT SUCCESS LOOKS LIKE

- Hooks, builds, drops, and quiet sections should feel intentional.
- Audit score should stay high with low overlap and low clutter.
- Neighbor props should react in a believable spatial flow.
- Report and notes files should describe a polished, ready result rather than a cleanup task list.
- If variants are enabled, the best output should be easy to identify or automatically promoted.

OPTIONAL REVIEW PATH

If you still want to inspect inside xLights:

- open the final `.xsq`
- inspect timing tracks to understand beats, stems, hooks, and section changes
- make optional artistic adjustments only where you want a personal touch

That review is now optional. It is not the core workflow.

TROUBLESHOOTING

- If no template is found, place `template.xsq` in the workspace or pass `--template`.
- If layout targeting looks wrong, refresh `xlights_rgbeffects.xml` or `xlights_rgbeffects.xbkp`.
- If you want the engine to mimic your favorite prior work better, place those `.xsq` files in the workspace history folder and rerun with `--learn-from-my-xsqs`.
- If Moises is unavailable, the engine falls back to local stem analysis automatically.
- If you want the engine to compare multiple artistic directions before choosing one, keep `--variants 3 --auto-shortlist` enabled.
